# 零成本持仓机 · A股资产分批配置与成本管理工具

> 面向中国 A 股和基金自持的个人量化记账工具。以「分批建仓 → 回收本金 → 零成本留存」为核心流程，内置 PE/PB 估值百分位、ATR / 网格收割算法，并将排雷、建仓、记账、收割四个环节以标的为轴心全程串联。
>
> **无后端 · 无登录 · 不触达执行 · 数据全部存本机**

---

## 设计理念

本工具将「分批建仓、逐步回收本金、零成本持有剩余仓位」这一广泛流传的个人理财思路数字化，提供可计算、可记录、可复盘的完整工具链。所有代码实现、算法设计与交互逻辑均为独立原创开发。

| 阶段 | 动作 | 目标 |
|:----:|------|------|
| 🔍 **筛选** | 评估估值百分位与财务质量，硬性过滤风险标的 | 只对基本面清晰的标的建仓 |
| 🌱 **建仓** | 按预设价位分批配置，不临盘追单 | 摊低持仓成本 |
| 💧 **管理** | ATR / 网格计算高抛低吸价位，持续降低成本 | 用价格波动为仓位"减负" |
| 📊 **回收** | 触达目标价时精确计算最优卖出数量 | 卖出 N 股，让剩余 M 股成本归零 |
| 🏆 **留存** | 本金全部回收后保留剩余仓位 | 剩余持仓等同于零成本持有 |

## 流程串联

各模块以**同一只标的**为轴心串联，跨页面自动传递数据，无需手动搬运：

```
① 排雷筛选
   输入代码或名称
   → 自动拉取 PE/PB 历史百分位、K 线趋势、财务质量、分红记录
   → 综合评分 0~100，五档结论
   → 评分通过后点「制定建仓计划」
                        ↓
② 建仓计划
   自动带入估值数据和当前价
   → 顶部三色仓位建议（低位绿 / 中性金 / 高位红）
   → 生成分批方案表格
   → 每批卡片「记录此批入账」/ 底部「记录第一批入账」
                        ↓
③ 持仓账本
   自动预填代码、名称、资产类型、价格、数量
   → 一键保存入账
   → 持仓卡片三个快捷跳转：
       ├─ 「计算收割」─ 自动带入剩余成本和持仓数量
       ├─ 「继续建仓」─ 带入当前成本价，制定补仓计划
       └─ 「重新筛选」─ 带入代码，更新最新估值状态
                        ↓
④ 收割计算
   自动填充剩余本金缺口和持仓数量
   → 网格 / ATR 双模式计算高抛低吸区间
   → 满足条件时高亮显示精确提示：
     ⚡ 卖出 1,200 股（¥12.50），即可让剩余 3,800 股成本降至 0 元
   → 「记录回收」写回持仓账本
                        ↓
⑤ 零成本资产库
   已回收资金 ≥ 总投入成本时自动进入
   → 仍有持仓：展示净收益持有状态
   → 已全部减仓：显示「回本毕业」
```

## 功能模块

### 导航结构（5 个底部标签）

| 标签 | 内容 |
|------|------|
| **计划** | 建仓计划 + 收割计算（顶部双 Tab 切换，状态独立保持）|
| **自选** | 添加 / 删除自选股，实时行情刷新，左滑删除；可为每只股设置目标价与警戒价 |
| **持仓** | 持仓账本 + 每张卡片三键快捷跳转 |
| **排雷** | 自动估值筛选；通过后一键跳转建仓计划 |
| **策略** | 筛选、建仓、回本、留存的核心框架与退出红线 |

### 建仓计划

- 输入可用本金、首批价格 / 净值、批数、下行间距、回收幅度，自动推演分批方案
- 从排雷页或持仓卡片跳入时，代码、名称、价格自动预填，并展示 PE 百分位三色建议
- 每批方案卡片内置「记录此批入账」，底部「记录第一批入账」按钮直连入账页并预填所有字段

### 收割计算

- **网格模式**：以固定百分比间距计算上下轨
- **ATR 模式**：自动从 K 线数据计算真实波幅（Wilder ATR，标准 14 日周期），也可手动输入；以 ATR × 倍数计算动态区间
- 从持仓跳入时，剩余本金缺口和持仓数量自动填充
- 零成本可达时，以金边高亮卡片输出精确提示语，含标的名称、卖出数量、触发价、剩余归零数量

### 排雷筛选

- 输入六位代码或中文名，自动拉取：当前价、PE / PB 历史百分位、K 线趋势、ST 状态、质押率、负债率、商誉占比、现金流利润比、股息率、连续分红年数
- 硬性过滤（一票否决）：ST / \*ST、退市风险、违规担保、财务造假、质押率 ≥ 50%、经营现金流为负
- 综合评分 0～100，五档结论：`适合建仓` / `可小额建仓` / `谨慎观察` / `暂不建仓` / `禁止建仓`
- 非禁止档时底部出现「制定建仓计划」按钮，携带估值上下文跳转

### 持仓账本

- 股票（股数）和基金（份额）混合记账
- 每个标的支持多批次，自动汇总本金回收进度、零成本状态、剩余数量
- 支持截图 OCR 文字粘贴解析快速入账
- 支持一键生成账单文本并复制导出

### 零成本资产库

- 已回收资金 ≥ 总投入成本时自动归入
- 仍有剩余持仓：展示净收益持有状态与剩余数量
- 已全部减仓：标记「回本毕业」

## 零成本账务公式

以下为本工具独立实现的账务核算逻辑：

```
总投入成本   = Σ（配置价格 × 数量 + 佣金手续费）
已回收资金   = Σ（回收价格 × 回收数量）+ Σ 现金分红 / 派发
零成本进度   = 已回收资金 ÷ 总投入成本            （0% ～ 100%）
零成本达成   = 已回收资金 ≥ 总投入成本
剩余成本价   = max(总投入成本 − 已回收资金, 0) ÷ 剩余数量
净收益持仓量 = 零成本达成后的剩余股数 / 份额
```

## 技术栈

| 层级 | 选型 |
|------|------|
| 跨端框架 | Flutter 3.x · iOS + Android |
| 状态管理 | Riverpod 2.x（riverpod_annotation 代码生成）|
| 本地数据库 | SQLite（sqflite）|

| 行情数据 | 新浪财经（主） / 腾讯财经（备） / 东方财富（搜索、指数、北向）/ 聚合数据（可选，需 key）|

| 统计图表 | fl_chart（估值雷达、百分位柱图）|
| ATR 算法 | 本地 Wilder ATR，从 K 线自动计算，离线可用 |
| 价格提醒 | flutter_local_notifications（系统推送，支持 Android / iOS）|
| 轻量存储 | shared_preferences（偏好设置）|
| UI 组件 | shimmer / flutter_slidable / percent_indicator / badges / iconsax |

## 快速开始

### 1. 安装 Flutter

```bash
# 官方文档：https://flutter.dev/docs/get-started/install/macos
# 安装完成后将 flutter 加入 PATH
echo 'export PATH="$PATH:$HOME/development/flutter/bin"' >> ~/.zshrc
source ~/.zshrc

# 验证环境
flutter doctor
```

### 2. 克隆并运行

```bash
git clone <repo-url>
cd stock-seed-app

flutter pub get        # 安装依赖
flutter run            # 运行到已连接的模拟器或真机
```

### 3. 构建产物

```bash
flutter build apk --release      # Android APK
flutter build ios --release      # iOS（需要 Xcode）
```
